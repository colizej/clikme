# SEO-стратегия — clikme.ru

## Текущие активы (март 2026)

- ~70 статей, позиции 1-10 по запросам о Вьетнаме/Нячанге
- Домен clikme.ru — возраст + ссылочная масса
- Googlebot активно посещает сайт (подтверждено логами)
- Семантическое ядро: рестораны, туризм, жильё, достопримечательности Нячанга

## Правила которые нельзя нарушать при миграции

### 1. URL — неприкосновенны
```
✅ /vizaran-prodlenie-prebyvaniya-vo-vetname       ← оставить точно так
✅ /blog-soveti-history/romanticheskiy-uzhin-nyachang
❌ /blog/vizaran-prodlenie-prebyvaniya-vo-vetname  ← менять нельзя
❌ /articles/2024/vizaran...                       ← менять нельзя
```

Если URL всё же меняется — **обязательно 301-редирект** со старого на новый.

### 2. Title и Description — копировать дословно
Не "улучшать" при миграции — сначала перенести как есть, потом оптимизировать.

### 3. Canonical — на каждой странице
```html
<link rel="canonical" href="https://clikme.ru/{{ article.slug }}/">
```

### 4. sitemap.xml — обновлять и отправлять
- Генерировать через `django.contrib.sitemaps`
- Отправлять после каждой новой статьи в GSC и Яндекс.Вебмастер

## Технический SEO-чеклист Django

### Обязательно в Фазе 1

- [ ] `<title>` — уникальный для каждой страницы
- [ ] `<meta name="description">` — уникальный, 120-160 символов
- [ ] `<link rel="canonical">` — на каждой странице
- [ ] `robots.txt` — разрешить всё полезное, закрыть admin/
- [ ] `sitemap.xml` — все публичные статьи и категории
- [ ] Редиректы: www → non-www (или наоборот, консистентно)
- [ ] HTTPS — обязательно
- [ ] Скорость: сжатие gzip, кэш статики, оптимизация изображений
- [ ] Open Graph теги — для Telegram и ВКонтакте (важно для РФ аудитории)
- [ ] Hreflang — пока `ru`, при добавлении языков расширить

### robots.txt
```
User-agent: *
Allow: /
Disallow: /admin/
Disallow: /api/
Disallow: /accounts/

Sitemap: https://clikme.ru/sitemap.xml
```

### Структурированные данные (JSON-LD)

```python
# Для статьи блога
ARTICLE_SCHEMA = {
    "@context": "https://schema.org",
    "@type": "Article",
    "headline": "...",
    "description": "...",
    "image": "...",
    "datePublished": "...",
    "dateModified": "...",
    "author": {"@type": "Person", "name": "..."},
    "publisher": {
        "@type": "Organization",
        "name": "КликМи",
        "url": "https://clikme.ru"
    }
}

# Для локального бизнеса (каталог мест — Фаза 2)
PLACE_SCHEMA = {
    "@context": "https://schema.org",
    "@type": "Restaurant",  # или TouristAttraction, LodgingBusiness
    "name": "...",
    "address": {"@type": "PostalAddress", "addressLocality": "Нячанг"},
    "geo": {"@type": "GeoCoordinates", "latitude": "...", "longitude": "..."}
}
```

## Мониторинг позиций

### Инструменты
- **Google Search Console** — бесплатно, основной
- **Яндекс.Вебмастер** — важен для РФ аудитории
- **Topvisor / Serpstat** — платно, для регулярного трекинга

### Что отслеживать
1. После миграции: ежедневно первые 2 недели
2. После переключения DNS: сравнение с baseline
3. Ключевые статьи с позиций 1-3: еженедельно

### Baseline (зафиксировать ДО миграции)
Скопировать из GSC топ-20 страниц с позициями в файл:
`docs/seo_baseline_[дата].csv`
